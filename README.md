# Pellets Analyzer — Интеллектуальная экосистема анализа биотоплива

[![Python 3.11+](https://img.shields.io/badge/python-3.11+-blue.svg)](https://www.python.org/downloads/)
[![Flask 3.1.2](https://img.shields.io/badge/flask-3.1.2-lightgrey.svg)](https://flask.palletsprojects.com/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Docker](https://img.shields.io/badge/docker-ready-blue.svg)](https://www.docker.com/)

**Pellets Analyzer** — аналитическая платформа на базе Flask для комплексного исследования, визуализации и оптимизации характеристик топливных пеллет. Система объединяет управление лабораторными данными, прогнозирование на базе машинного обучения и модуль бизнес-аналитики.

## 🎯 Возможности

### Основные функции
| Функция | Описание |
|---------|----------|
| **Сбор данных** | Импорт из Excel/CSV, консолидация в единую SQLite-базу |

## 🤖 Модуль машинного обучения

### Прогнозирование (Predictor)
- **Алгоритмы**: Random Forest, Gradient Boosting (XGBoost)
- **Автоподбор**: GridSearchCV для оптимизации гиперпараметров
- **Cold-Start**: Линейная аппроксимация при недостатке данных
- **Параллелизм**: Многопоточное обучение для разных свойств

### Оптимизация (Optimizer)
- **Метод**: SLSQP (последовательное квадратичное программирование)
- **Ограничения**: Целевые показатели, доступность компонентов, бюджет
- **Результат**: Оптимальный состав с прогнозируемыми свойствами

## 🔐 Аутентификация и безопасность

### Система аутентификации
- Регистрация с подтверждением email
- Вход с защитой от brute-force (блокировка после 5 попыток)
- Восстановление пароля через email
- Сессии с безопасным хранением
- Роли: пользователь / администратор

### Защита от атак
| Тип атаки | Механизм защиты |
|-----------|-----------------|
| SQL Injection | Параметризованные запросы + валидация |
| XSS | Санитизация входных данных |
| CSRF | Токены для POST-запросов |
| Brute Force | Блокировка после 5 неудачных попыток |
| Rate Limiting | Ограничение частоты запросов |
| File Upload | Валидация расширений и размера файлов |
| Clickjacking | X-Frame-Options заголовок |

Подробнее: [`security.py`](security.py), [`DEPLOY.md`](DEPLOY.md)

## 🚀 Быстрый старт

### Требования
- **Python 3.11+** ([скачать](https://www.python.org/downloads/))
- **Windows 10/11**, **Linux** или **macOS**
- **2 GB RAM** минимум, **4 GB** рекомендуется

### Способ 1: Автоматическая установка (Windows)
```cmd
install.bat
```
Скрипт автоматически:
1. Проверит наличие Python
2. Создаст виртуальное окружение
3. Установит все зависимости
4. Запустит приложение

### Способ 2: Ручная установка
```bash
# 1. Создание виртуального окружения
python -m venv venv

# 2. Активация
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

# 3. Установка зависимостей
pip install -r requirements.txt

# 4. Настройка конфигурации
copy mail_config.example.json mail_config.json
# Отредактируйте mail_config.json, указав свои SMTP-данные

# 5. Запуск
python main.py
```

Приложение доступно по адресу: **http://localhost:5000/**

## 🐳 Docker-развёртывание

### Быстрый запуск
```bash
# 1. Клонирование
git clone <repository-url>
cd Pellets_Analyzer

# 2. Настройка
cp .env.example .env
# Отредактируйте .env, указав SECRET_KEY и SMTP-данные

# 3. Запуск
docker-compose up -d

# 4. Проверка
docker-compose ps
docker-compose logs -f web
```

### Docker-команды
| Команда | Описание |
|---------|----------|
| `docker-compose up -d` | Запуск в фоновом режиме |
| `docker-compose down` | Остановка и удаление контейнеров |
| `docker-compose logs -f` | Просмотр логов |
| `docker-compose up -d --build` | Пересборка и перезапуск |
| `docker-compose exec web bash` | Вход в контейнер |

### Структура docker-compose
- **web** — основное приложение (gunicorn, 4 воркера)
- **volumes** — сохранение данных между перезапусками
- **healthcheck** — автоматический мониторинг здоровья

Подробнее: [`Dockerfile`](Dockerfile), [`docker-compose.yml`](docker-compose.yml)

## 📧 Настройка email

### Gmail
1. Включите **2-Step Verification** в Google Account
2. Создайте **App Password**: Account → Security → App Passwords
3. Скопируйте [`mail_config.example.json`](mail_config.example.json) → [`mail_config.json`](mail_config.json)
4. Укажите email и App Password в [`mail_config.json`](mail_config.json)

## 🛡 Безопасность продакшена

### Nginx (обратный прокси)
- SSL/TLS (TLSv1.2+)
- Rate limiting на уровне сервера
- Security заголовки (CSP, HSTS, X-Frame-Options)
- Блокировка доступа к конфигам

### Серверные меры
```bash
# Fail2Ban — блокировка подозрительных IP
sudo apt install fail2ban

# UFW Firewall
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# SSL сертификат (Let's Encrypt)
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```
