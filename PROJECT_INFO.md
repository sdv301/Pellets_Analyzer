# Документация проекта Pellets Analyzer

## 1. Что делает приложение (Назначение)
**Pellets Analyzer** — это специализированная аналитическая платформа для производителей и исследователей биотоплива (пеллет). Основная цель программного комплекса:
*   **Сбор и систематизация данных**: Перевод разрозненных Excel-отчетов из лабораторий в единую защищенную базу данных SQLite.
*   **Прогнозирование качества**: Предсказание характеристик будущих партий пеллет (теплота сгорания, прочность, зольность) еще до этапа производства на основе их химического и компонентного состава.
*   **Оптимизация рецептуры**: Поиск математически идеального соотношения компонентов (например, сколько добавить торфа в опилки), чтобы получить продукт с заданными свойствами при минимальных затратах.
*   **Интеллектуальный аудит**: Выявление скрытых зависимостей и трендов в данных с помощью алгоритмов машинного обучения и ИИ.

---

## 2. Какие технологии использует (Стек)
*   **Язык**: Python 3.8+
*   **Веб-каркас**: Flask (маршрутизация, сессии, шаблонизация Jinja2).
*   **База данных**: SQLite (локальное хранилище данных).
*   **Анализ данных**: 
    *   `Pandas` — обработка таблиц.
    *   `NumPy` — математические вычисления.
    *   `SciPy` — алгоритмы математической оптимизации (SLSQP).
*   **Машинное обучение (ML)**: 
    *   `Scikit-learn` — модели Random Forest и Gradient Boosting для предсказания свойств.
*   **Визуализация**: 
    *   `Plotly` — интерактивные графики в браузере.
    *   `Matplotlib / Seaborn` — классические статистические графики.
*   **Frontend**: HTML5, Vanilla CSS (без тяжелых библиотек), JavaScript для динамического обновления данных без перезагрузки страниц.

## 3. Описание страниц и их функций

### 🏠 Главная страница (Index)
*   **Функция**: Точка входа и первичная загрузка данных.
*   **Возможности**: Загрузка новых файлов CSV/Excel в систему, предварительный просмотр содержимого базы данных, удаление старых записей.

### 📋 Таблицы (Tables)
*   **Функция**: Работа с накопленными данными.
*   **Возможности**: Полнотекстовый поиск по составам, фильтрация по параметрам, пагинация (постраничный просмотр) больших объемов данных.

### 📊 Создание графиков (Create Graph)
*   **Функция**: Инструмент визуального анализа.
*   **Возможности**: Выбор типов графиков (Scatter, Boxplot, 3D), настройка осей X/Y/Z, фильтрация данных для отображения, экспорт готовых изображений. Позволяет увидеть зависимости, например, как зольность влияет на температуру выгорания.

### 🔄 Сравнительная таблица (Compare)
*   **Функция**: Глубокое сопоставление образцов.
*   **Возможности**: Выбор до 5 конкретных составов и их детальное сравнение по критериям (теплотехника, механика, выбросы). Подсветка значимых различий между составами.

### 🤖 ML Анализ / Дашборд (ML Dashboard)
*   **Функция**: Центр управления искусственным интеллектом.
*   **Возможности**: 
    *   **Обучение моделей**: Запуск обучения на текущих данных.
    *   **Предсказание свойств**: Ввод нового состава и получение прогноза его характеристик.
    *   **Оптимизация состава**: Указываете цель (например, "макс. теплота сгорания"), и система сама высчитывает нужный процент компонентов.

### 🧠 ИИ-Анализ (AI Analysis)
*   **Функция**: Чат-интерфейс для работы с данными.
*   **Возможности**: Запрос анализа на естественном языке (например, "Какой состав самый экологичный?"). Система анализирует данные и дает текстовое заключение.

### 📜 История ML (ML History & Saved Models)
*   **Функция**: Аудит и управление моделями.
*   **Возможности**: Просмотр точности обученных моделей (R², ошибка MAE) и управление версиями сохраненных алгоритмов.

## 4. Зачем всё это нужно бизнесу?
1.  **Снижение брака**: Система предупреждает, если планируемый состав не пройдет по ГОСТу.
2.  **Экономия сырья**: Находит способ заменить дорогие компоненты дешевыми аналогами без потери качества.
3.  **Скорость**: Лабораторный расчет, который раньше занимал часы, система делает за миллисекунды.
4.  **Data-Driven решения**: Уход от интуитивного производства к научно обоснованному.
