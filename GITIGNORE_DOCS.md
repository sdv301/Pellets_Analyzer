# 📁 GitIgnore Documentation

Этот файл описывает, какие файлы и папки игнорируются в проекте Pellets Analyzer.

## ✅ Что игнорируется

### 🔹 Python-артефакты
- `__pycache__/` - кэшированные файлы Python
- `*.py[cod]`, `*$py.class` - скомпилированные файлы
- `*.so` - C расширения
- `*.egg-info/` - информация о пакетах

### 🔹 Виртуальные окружения
- `.venv/`, `venv/`, `env/` - виртуальные окружения
- `.env` - файлы окружения (могут содержать секреты)

### 🔹 Базы данных
- `*.db`, `*.sqlite`, `*.sqlite3` - файлы баз данных
- `pellets_data.db` - основная БД проекта

### 🔹 Пользовательские файлы
- `Uploads/` - загруженные пользователями файлы
- `sessions/` - файлы сессий Flask
- `*.log`, `app_errors.log` - логи приложения

### 🔹 Тестовые файлы
- `test_*.py`, `*_test.py` - тесты
- `tmp_*.py`, `temp_*.py` - временные скрипты
- `parse_excel_*.py` - скрипты для парсинга

### 🔹 Резервные копии
- `*_old.py`, `*_backup.py` - старые версии файлов
- `*.bak`, `*.backup` - файлы резервных копий

### 🔹 ML модели
- `models/` - папка с моделями
- `*.joblib`, `*.pkl` - сериализованные модели

### 🔹 Документы
- `*.docx`, `*.xlsx`, `*.xls` - документы Office (кроме requirements.xlsx)
- `IMPROVEMENTS_SUMMARY.md` - временная документация

### 🔹 IDE и редакторы
- `.idea/`, `.vscode/` - настройки IDE
- `*.iml`, `*.code-workspace` - файлы проектов

### 🔹 ОС
- `.DS_Store`, `Thumbs.db` - системные файлы ОС

## ❌ Что НЕ игнорируется (важно!)

- `main.py` - основной файл приложения
- `requirements.txt` - зависимости Python
- `templates/*.html` - HTML шаблоны
- `static/` - статические файлы (CSS, JS, изображения)
- `database.py`, `data_processor.py` - основные модули
- `README.md` - документация проекта

## 📝 Примечания

1. **Конфигурационные файлы**: `config.py` игнорируется, так как может содержать чувствительные данные. Создайте `config.example.py` с шаблонными значениями.

2. **Базы данных**: Файлы БД игнорируются, но структура БД должна быть описана в документации или миграциях.

3. **Загруженные файлы**: Папка `Uploads/` игнорируется, так как содержит пользовательские данные.

4. **ML модели**: Большие файлы моделей игнорируются. Используйте Hugging Face или другое хранилище для моделей.

## 🔄 Как добавить файл в .gitignore

Если файл уже отслеживается, но вы хотите его игнорировать:

```bash
# Удалить файл из индексации Git (но оставить на диске)
git rm --r --cached <файл>

# Добавить изменения
git add .gitignore

# Закоммитить
git commit -m "Add <файл> to .gitignore"
```
